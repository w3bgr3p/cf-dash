# Cloudflare для разработчика — что реально можно сделать бесплатно

Большинство людей знают Cloudflare как "штука которая защищает сайты от DDoS и делает их быстрее". Это правда, но это 10% от того что там есть. За последние три года Cloudflare превратился в полноценную платформу для деплоя приложений — и большую часть этого можно использовать бесплатно.

Вот что реально даёт один бесплатный аккаунт и что на нём можно построить.

---

## Что входит в бесплатный тир

### Workers — serverless без сервера

100 000 запросов в день. Бесплатно. Без кредитной карты.

Это serverless функции которые запускаются в 330+ городах мира с холодным стартом меньше 1 миллисекунды. Пишешь JavaScript, TypeScript, Python или Rust — деплоишь одной командой `wrangler deploy` и получаешь URL вида `my-worker.account.workers.dev`.

100К запросов в день — это не мало. Для личного инструмента, небольшого SaaS, webhook обработчика или API прокси этого хватает с запасом. Даже если каждый запрос это реальный пользователь — это больше 3 миллионов запросов в месяц.

**Что важно:** нет платы за трафик. Совсем. Это принципиальное отличие от AWS Lambda где исходящий трафик стоит $0.09 за гигабайт.

### D1 — SQLite в облаке

5 гигабайт хранилища, 5 миллионов операций чтения и 5 миллионов записей в месяц. Это полноценная SQL база данных, синтаксис SQLite, доступна из Worker через binding — без HTTP запросов, без задержки сети, просто `env.DB.prepare("SELECT...").all()`.

Для большинства личных проектов и небольших продуктов этого более чем достаточно.

### R2 — хранилище файлов без платы за трафик

10 гигабайт хранилища, 1 миллион операций записи и 10 миллионов чтений в месяц. S3-совместимый API — работают все существующие S3 SDK.

Главное отличие от AWS S3: **нулевая плата за исходящий трафик**. На S3 ты платишь $0.09 за каждый гигабайт который скачивают пользователи. На R2 — ноль. Для хранения медиафайлов, бэкапов, статики это огромная разница.

### KV — глобальный key-value стор

1 гигабайт данных, 100 000 операций чтения и 1000 записей в день. Данные реплицируются глобально — чтение происходит из ближайшего датацентра, это очень быстро.

Хорошо подходит для: конфигурация приложения, feature flags, кэширование API ответов, сессии пользователей.

### Durable Objects — когда нужно состояние

400 000 GB-секунд и 1 миллион запросов в месяц. Это Workers у которых есть персистентное состояние и которые существуют как единственный инстанс в конкретном датацентре.

Простой пример зачем это нужно: счётчик просмотров. Обычный Worker stateless — если два запроса пришли одновременно, оба прочитали "100" и оба записали "101" вместо "102". Durable Object решает это — он один и обрабатывает запросы последовательно.

Используется для WebSocket чатов, совместного редактирования, rate limiting, игровых сессий.

### Queues — асинхронные задачи

10 000 операций в день. Worker кладёт задачу в очередь, другой Worker забирает и обрабатывает. Гарантированная доставка, автоматические retry.

Без этого пришлось бы поднимать RabbitMQ или Redis — отдельный сервер, деньги, обслуживание. Здесь это просто binding в конфиге.

### Workers AI — GPU inference бесплатно

10 000 Neurons в день. Это единица вычислений на их GPU. Сбрасывается каждый день в полночь по UTC.

Что можно сделать за 10К neurons:
- ~50-200 запросов к LLM (Llama 4, Gemma, Mistral, DeepSeek)
- ~33 минуты транскрипции аудио через Deepgram Nova-3
- тысячи embedding векторов через BGE-M3

Cloudflare держит GPU в своих датацентрах и даёт к ним доступ. Один HTTP запрос с Bearer токеном — и у тебя inference на GPU без аренды GPU сервера.

## Что реально можно построить на одном аккаунте

### Полноценный backend для приложения

Workers + D1 + R2 — достаточно для небольшого продукта. REST API на Workers, данные в D1, файлы в R2. Без единого сервера, без DevOps, без счётов за EC2.

Деплой занимает секунды. Масштабирование автоматическое.

### Webhook обработчик для любого сервиса

Telegram бот, GitHub webhook, Stripe webhook, Slack бот — всё это принимается Worker'ом. Верификация подписи, обработка, ответ за миллисекунды. Если обработка долгая — кладёшь в Queue и отвечаешь 200 немедленно.

### AI pipeline для Telegram

Голосовые сообщения → транскрипция через Nova-3 (точнее Whisper по WER) → LLM для анализа → ответ или сохранение в D1. Всё на бесплатных ресурсах CF, без единого сервера.

### Персональный CDN без платы за трафик

R2 + Workers = раздача файлов глобально. Нет лимита на скачивание. Для сравнения: AWS CloudFront берёт за трафик, Cloudflare R2 — нет.

### Cron задачи без cron сервера

В `wrangler.toml` указываешь расписание — Worker запускается автоматически. Ежедневный парсинг, еженедельные отчёты, периодическая очистка базы — без crontab и без сервера который нужно держать включённым.

### Прокси и middleware для любого трафика

Rate limiting, auth проверка, логирование, A/B тестирование, гео-блокировка — делается в несколько строк кода в Worker'е и применяется ко всему трафику глобально.

### RAG система для личной базы знаний

Embeddings через BGE-M3 (бесплатно на Workers AI) + pgvector = семантический поиск по своим документам. Спрашиваешь "какой сервис подходит для очереди задач" — находит нужную карточку с описанием, лимитами, когда использовать и когда нет.

### Multi-agent система разработки

Workers как orchestrator — принимает задачу, раздаёт агентам через Service Bindings, собирает результат, пишет в Jira через API. Полная история в Jira, никакого своего сервера.

## Почему это интересно именно сейчас

Раньше "без сервера" означало AWS Lambda + API Gateway + S3 + RDS — каждый компонент отдельно настраивается, отдельно биллится, отдельно мониторится. Сложно и дорого для небольших проектов.

Cloudflare собрал всё в одно место с единой моделью биллинга и единым конфигом (`wrangler.toml`). Compute, storage, database, queues, AI — всё это bindings в одном файле.

Для разработчика который хочет быстро запустить идею или инструмент — это меняет уравнение. Не "аренда сервера + настройка + обслуживание", а "написал код → задеплоил → работает".

## Где платный план реально нужен

Бесплатный тир заканчивается когда:

| Сервис | Бесплатно | Платно |
|---|---|---|
| Workers | 100K req/day | $5/мес → 10M req/мес |
| Workers AI | 10K neurons/day | $0.011 за 1K neurons |
| D1 | 5GB, 5M ops/мес | по использованию |
| R2 | 10GB, 1M writes/мес | $0.015/GB/мес |
| KV | 1GB, 100K reads/day | по использованию |

Workers Paid за $5 в месяц даёт 10 миллионов запросов вместо 100К в день. Для большинства реальных продуктов этого достаточно.

## Итог

Один бесплатный аккаунт Cloudflare это:

- Serverless backend — 100K req/day, 330+ городов, <1ms cold start
- SQL база данных — 5GB, без настройки, прямо из кода
- Файловое хранилище — 10GB, нулевой egress трафик
- Глобальный KV кэш — 1GB, реплицирован везде
- Очереди сообщений — async обработка без RabbitMQ
- AI inference — LLM, STT, embeddings на реальных GPU
- Cron задачи — без сервера
- Всё с деплоем в одну команду

Для личных инструментов, автоматизации, небольших продуктов и прототипов — это полноценная инфраструктура. Бесплатно.
